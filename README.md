# City Guardian - A Public Infrastructure Issue Reporting System

A comprehensive digital platform designed to bridge the communication gap between citizens and municipal authorities[cite: 5, 6]. Citizens can report real-world community issues such as broken streetlights, potholes, water loggings, and waste mismanagement, while administrators and assigned staff handle verification, tracking, and sequential resolution workflows.

## ✨ Key Features

* **Role-Based Dynamic Dashboards:** Custom tailored, private dashboard views and permission layers restricted to Admins, Staff, and Citizens.

* **Real-time Comprehensive CRUD Operations:** Citizens can effortlessly submit new public reports, modify pending items using pre-filled inline modals, or remove records seamlessly.

* **Interactive Issue Timeline Tracking:** A chronological, read-only vertical stepper UI displaying structural update entries like creation, assignment, and step-by-step state progress to maximize procedural transparency.

* **Algorithmic Priority Sorting:** Dynamic query sorting logic ensuring that high-priority and critical infrastructure issues permanently anchor at the top of tabular queues and feed views.

* **Single-Vote Community Endorsement:** Authenticated users can upvote existing local community entries exactly once to establish public importance, updating metrics dynamically across client views while blocking users from upvoting self-submitted issues.

* **Automated Account Status Restrictions:** Safety guardrails that automatically disable form submissions, report editing, and community upvoting privileges instantly for accounts flagged or blocked by administration panels.

* **Administrative Staff Provisioning & Management:** Fully isolated administrative modules allowing secure staff account creation synced directly across database layers and authentication platforms, paired with real-time update and removal workflows.

* **One-Click Targeted Allocation:** Streamlined admin interfaces allowing direct, absolute staff assignment for unresolved public tickets through responsive modal interfaces.

* **Sequential Resolution Workflows:** Specialized operational pipelines allowing assigned personnel to advance issues iteratively through structured operational lifecycles (Pending $\rightarrow$ In-Progress $\rightarrow$ Working $\rightarrow$ Resolved $\rightarrow$ Closed).

* **Data-Driven Visual Analytics:** Beautiful overview metrics featuring statistical summaries, dynamic calculation card sets, and chart distributions highlighting performance parameters across all layout dashboards.

* **Persistent Session Maintenance:** Bulletproof private route structures utilizing TanStack Query synchronization to ensure active authorization states remain uninterrupted upon hard page refreshes.

* **Responsive Fluid Design:** A fully responsive user interface adaptive to mobile viewports, tablet environments, and desktop displays built using custom design structures.
